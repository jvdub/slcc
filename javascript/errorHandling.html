Error Handling

For when things don't go exactly as planned.

For your website to perform well under all circumstances, you'll need to tell it how to handle errors.

Your users will inevitably do things that you didn't think about. Adding code to handle errors is essential so that the website continues to operate when something happens that it doesn't expect.

If you don't handle errors, your site could be partially impaired or completely unusable.

Exceptions

Most programming languages (JavaScript included) have a concept of Exceptions.

Exceptions are what happen when the program encounters an error.

You might say the code you write are the "rules" the computer follows and when something breaks those rules, it's en "exception" and you'll need to tell it how to handle it.

Exceptions can happen for a number of reasons. Here are some examples:

  • The code syntax is invalid
  • The object property or method doesn't exist
  • The variable isn't of the data type you thought it was
  • Something went wrong with an HTTP request
  • ... and many, many more ...

Handling Exceptions

All programming languages have a way to deal with errors as they arise.

JavaScript (and many other languages) use the try and catch keywords to handle exceptions.


                    try {
                        // Do a thing
                    } catch (e) {
                        // Something went wrong while doing a thing
                        // The variable `e` usually has a message and other helpful things to figure out what went wrong.
                    }

Here's a more real example:


                    function calculateSlope(rise, run) {
                        return rise / run;
                    }

                    let x = 0;
                    let y = 5;

                    try {
                        let slope = calculateSlope(y, x);
                    } catch (e) {
                        console.log('Something went wrong!');
                        console.log(e);
                    }

In the code above, a divide by zero exception is thrown. We catch it and log it to the console.

Because we catch it, the website keeps operating. If we didn't catch it, the site would stop working.

Every once in a while, you want to do something no matter if an exception occurs or not.

You can do this with the finally keyword.


                    try {
                        // Do a thing
                    } catch (e) {
                        // Something went wrong while doing a thing
                        // The variable `e` usually has a message and other helpful things to figure out what went wrong.
                    } finally {
                        // This runs whether or not an error was thrown.
                    }

The finally block is usually used for clean-up items such as closing a file, closing a connection, deleting things that aren't needed, or other clea-up tasks.

Throwing Exceptions

From time to time, you'll encounter situations where you want to raise an exception. You want to create an error.

You can do this using the throw keyword.

If you ever throw and exception, you should always catch it later if possible.


                    function calculateSlope(rise, run) {
                        if (run === 0) {
                            throw 'Run cannot be zero!';
                        }

                        return rise / run;
                    }

                    let x = 0;
                    let y = 5;

                    try {
                        let slope = calculateSlope(y, x);
                    } catch (e) {
                        console.log('Something went wrong!');
                        console.log(e);
                    }

This time our code shows "Run cannot be zero" instead of a divide by zero error.

Other Useful Debugging Tools

There are a few other things that can help you debug your site:

  • The debugger keyword will automatically stop execution on that line if the debugger is open.
  • console.log() will print just about anything to the developer console for you to inspect later.

Debugging is a skill. The more you do it, the more proficient you'll get. Many debugging skills take time to learn. If you get frustrated, don't give up. Hang in there and you'll figure it out.
