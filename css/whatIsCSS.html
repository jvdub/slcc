What is CSS?

CSS is the third part of a website:

Style

Now that we have the content and the structure for our resume sites, we can start improving how it looks.

We do that with CSS.

CSS Stands for Cascading Style Sheets.

It's called this because styles are applied in a certain order and "cascade" to lower levels.

Here are the different places styles can be applied in order of precedence:

  1. Inline in the HTML itself (takes precedence over all else with very few exceptions)
  2. Embedded Styles (declared in the <head> of the HTML file)
  3. External Style Sheets (this is the industry standard location)
  4. Browser Style Sheets (each user has control over these)

CSS files are often cached to help improve browser performance.

While helpful, this can often lead to frustrations in development.

Pro Tip: disable your cache or hard refresh every time you want to view your site.

Syntax

Syntax is the order of words and symbols that are valid in the language.

CSS is broken into selectors and declarations.

Each selector matches to a certain element or group of elements in the HTML file.

Each declaration has a property and a value that define how it should be displayed. 


                    selector { declaration; }
                

                    selector { property: value; }
                

                    selector {
                        property: value;
                        property: value;
                        property: value;
                    }

All declarations must be within curly braces. { ... }

Each property must be followed by a colon. Each value must be followed by a semi-colon.

Each selector with its accompanying declarations is loosely referred to as a Rule.

Let's chat about Selectors

Selectors map to HTML elements.

There are certain symbols that distinguish in which way it should match.

A selector with no symbol will match to an HTML element with that name.

A CSS selector of body will match to the <body> tag in the HTML page.

A CSS selector of p will match with all <p> tags in the HTML page.

The octothorpe (#) matches against id's.

A CSS selector of #skills-list will match to an element with the id skills-list in the HTML page.

The period (.) matches against classes.

A CSS selector of .container will match to all elements with a CSS class of container in the HTML page.

There are other symbols that can help you narrow in on very specific elements in specific use cases, but these are the basics that you'll need for now.
