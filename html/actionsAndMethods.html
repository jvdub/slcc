Actions and Methods

Actions and Methods define how the data looks and where it goes.

The action attribute defines where to send the data when the user submits the form.

Usually this is a file on the server that can process the data it receives (we'll use PHP for this in the last course)

The method attribute defines if the data is sent in the URL or "hidden" in the body of the request.

There are two main methods that are used; GET and POST

The GET method sends the data in the URL. This is useful for bookmarking requests but does make the data rather visible.

The POST method sends the data in the body of the request which makes the data less visible. This is generally used for when sensitive information or large amounts of data is being sent.

Here are some examples:


            <form action="doLogin.php" method="POST">
                <label>
                    Username:
                    <input type="text" name="username" value="">
                </label>
                <label>
                    Password:
                    <input type="password" name="password" value="">
                </label>
                <input type="submit" name="login" value="Log In">
            </form>
        

            <form action="addToNewsletter.php" method="GET">
                <label>
                    Email:
                    <input type="email" name="email" value="">
                </label>
                <input type="submit" name="signup" value="Sign Me Up!">
            </form>
